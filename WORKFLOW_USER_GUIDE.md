## 工作流程概覽

### 4 階段流程圖

```
Phase 1: Discovery (發現階段) - 3-5 分鐘
│
├── Master Workflow Agent → 需求分析
├── Framework Selector Agent → 框架推薦
└── Audience Analyzer Agent (可選) → 受眾分析
    ↓
Phase 2: Research (研究階段) - 8-12 分鐘
│
└── Interview Specialist Agent → 深度訪談
    ↓
Phase 3: Creation (創作階段) - 7-10 分鐘
│
└── 選定的 Framework Agent → 內容生成
    ↓
Phase 4: Optimization (優化階段) - 5-8 分鐘
│
├── Content Optimizer Agent → 內容優化
├── Platform Customizer Agent → 平台適配
└── Quality Assurance Agent → 品質檢查
```

### 所需 Agents 清單

**必需(MVP)**:
- Master Workflow Agent
- Framework Selector Agent
- Interview Specialist Agent
- 3 個 Framework Agents (SOLVE, BUSINESS, HEART)
- Content Optimizer Agent
- Quality Assurance Agent

**完整版**:
- 以上必需 Agents +
- Audience Analyzer Agent
- Content Gap Analyst Agent
- 7 個其他 Framework Agents
- Platform Customizer Agent

## Phase 1: Discovery (發現階段)

### 目標
分析用戶需求,選擇最適合的內容框架

### 使用的 Agents
1. Master Workflow Agent (必需)
2. Framework Selector Agent (必需)
3. Audience Analyzer Agent (可選)

---

### Step 1.1: Master Workflow Agent - 需求分析

#### 在哪裡操作
打開 "Master Workflow Agent" Project

#### 輸入模板

```
我想創作一篇內容,以下是我的背景資訊:

【個人背景】
- 職業/專業: [你的職業,如:數位行銷顧問]
- 經驗年資: [如:5年]
- 專業領域: [如:SEO, 內容行銷]

【創作需求】
- 主題: [想寫什麼,如:如何提升網站流量]
- 目標: [如:建立權威、增加互動、推廣產品]
- 目標受眾: [如:中小企業主、行銷人員]
- 主要平台: [如:LinkedIn]

【個人特質】
- 寫作風格偏好: [如:理性邏輯、感性故事、實務導向]
- 擅長的表達方式: [如:數據分析、案例分享、個人經驗]

請幫我分析並推薦最適合的內容創作框架。
```

#### 預期輸出

Master Workflow Agent 應該會提供:
1. 用戶特質分析
2. 目標分類
3. 推薦的框架方向
4. 下一步建議

#### 關鍵資訊提取

從回應中記錄:
- **用戶特質**: [如:實務導向型]
- **內容目標**: [如:知識分享]
- **推薦框架**: [如:SOLVE Method]

### 使用的 Agents
- Interview Specialist Agent (必需)

---

### Step 2.1: Interview Specialist Agent - 深度訪談

#### 在哪裡操作
打開 "Interview Specialist Agent" Project

#### 輸入模板

```
[背景資訊]
- 選定框架: [從 Phase 1 複製,如:SOLVE Method]
- 內容主題: [如:如何用 AI 工具提升內容產出效率]
- 目標受眾: [如:數位行銷從業者]
- 創作目標: [如:提供實用指南]

[我的經驗背景]
[簡單描述你的相關經驗,如:我從 2023 年開始使用 AI 工具輔助內容創作,嘗試了 10+ 種工具,建立了自己的 workflow,幫助團隊將內容產出效率提升了 3 倍]

請開始訪談我,幫我挖掘出創作這篇內容所需的素材。
```

#### 訪談過程

Interview Specialist Agent 會進行 5 階段訪談:

**階段 1: 暖場 (1-2 輪)**
- 簡單介紹和建立對話氛圍
- 你的回應:自然地分享背景

**階段 2: 核心事實 (3-5 輪)**
- 具體事實、數據、行動
- 問題示例:"具體是怎麼開始的?" "使用了哪些工具?" "結果如何?"
- 你的回應:提供具體細節和數據

**階段 3: 深度洞察 (3-4 輪)**
- 思考過程、動機、價值觀
- 問題示例:"為什麼做出這個決定?" "最大的挑戰是什麼?"
- 你的回應:分享深層思考和洞察

**階段 4: 情感故事 (2-3 輪)**
- 情感化的細節
- 問題示例:"印象最深的時刻?" "最困難的時候?"
- 你的回應:分享真實感受和情感經驗

**階段 5: 未來展望 (1-2 輪)**
- 前瞻性觀點
- 問題示例:"對未來的預測?" "給後來者的建議?"
- 你的回應:提供前瞻性觀點

#### 訪談技巧

✅ **好的回應**:
- 具體詳細,有數字和細節
- 真實坦誠,包括失敗和掙扎
- 有情感溫度,不只是事實陳述
- 包含獨特觀點和洞察

❌ **避免的回應**:
- 過於籠統("我做了很多努力")
- 只有表面資訊,沒有深度
- 迴避困難和挑戰
- 缺乏個人觀點

#### 預期輸出

訪談結束後,Interview Specialist Agent 會整理:
1. **核心故事線** (背景 → 衝突 → 行動 → 轉折 → 結果)
2. **關鍵洞察** (3-5 個核心觀點)
3. **情感時刻** (印象深刻的細節)
4. **實用建議** (可操作的建議)
5. **金句提取** (可引用的精彩語句)
6. **支撐數據** (具體數字和事實)

#### 關鍵資訊提取

將整理好的素材複製保存,包括:
- 核心故事線
- 關鍵洞察
- 金句
- 數據

## Phase 3: Creation (創作階段)

### 目標
使用選定的 Framework Agent 生成內容初稿

### 使用的 Agents
根據 Phase 1 選定的框架,使用對應的 Framework Agent

---

### Step 3.1: Framework Agent - 內容生成

#### 在哪裡操作
打開對應的 Framework Agent Project (如 "SOLVE Method Agent")

#### 輸入模板

```
[框架資訊]
- 使用框架: [如:SOLVE Method]
- 框架匹配度: [如:92分]

[內容需求]
- 主題: [如:如何用 AI 工具提升內容產出效率]
- 目標受眾: [如:數位行銷從業者]
- 目標平台: [如:LinkedIn]
- 預期長度: [如:800-1200字]

[訪談素材]
[從 Phase 2 複製所有整理好的素材]

核心故事線:
- 背景: [xxx]
- 衝突: [xxx]
- 行動: [xxx]
- 轉折: [xxx]
- 結果: [xxx]

關鍵洞察:
1. [xxx]
2. [xxx]
3. [xxx]

金句:
- "[xxx]"
- "[xxx]"

支撐數據:
- [xxx]
- [xxx]

請根據以上素材,使用 [框架名稱] 框架為我生成內容初稿。
```

#### 不同框架的重點提示

**SOLVE Method**:
- 強調問題定義和解決步驟
- 需要清晰的 step-by-step 指導

**BUSINESS Method**:
- 強調商業價值和 ROI
- 需要數據支撐和專業論證

**HEART Method**:
- 強調情感連結和故事性
- 需要真實脆弱的細節

#### 預期輸出

Framework Agent 應該會提供:
1. **完整的內容初稿** (按框架結構組織)
2. **框架執行檢查** (確認每個部分都完整)
3. **語言風格說明** (使用的語調和風格)

#### 內容初稿檢查

閱讀生成的內容,檢查:
- [ ] 結構完整,符合框架要求
- [ ] 使用了訪談素材
- [ ] 語言風格適合目標受眾
- [ ] 長度符合預期

如果不滿意,可以在同一個對話中要求修改:

```
內容整體不錯,但我希望調整以下部分:
1. [具體的修改要求]
2. [具體的修改要求]

請幫我修改。
```

## Phase 4: Optimization (優化階段)

### 目標
多維度優化內容,確保品質和效果

### 使用的 Agents
1. Content Optimizer Agent (必需)
2. Platform Customizer Agent (推薦)
3. Quality Assurance Agent (必需)

---

### Step 4.1: Content Optimizer Agent - 內容優化

#### 在哪裡操作
打開 "Content Optimizer Agent" Project

#### 輸入模板

```
[內容資訊]
- 框架: [如:SOLVE Method]
- 目標受眾: [如:數位行銷從業者]
- 目標平台: [如:LinkedIn]
- 創作目標: [如:建立權威,提供實用指南]

[內容初稿]
[從 Phase 3 複製完整內容]

請幫我診斷並優化這篇內容,重點關注:
1. 結構和邏輯流暢性
2. 語言可讀性和用詞精準度
3. 洞察深度和獨特觀點
4. 用戶體驗和行動引導

請提供具體的優化建議。
```

#### 預期輸出

Content Optimizer Agent 會提供:
1. **診斷報告** (識別的問題和改進機會)
2. **優化建議** (具體修改方案和優先級)
3. **改進版本** (可選:優化後的文本)

#### 應用優化建議

根據優化建議,你可以:
1. **自己修改**: 根據建議手動調整內容
2. **要求 Agent 修改**: 在對話中要求 Agent 應用建議並生成優化版本

```
謝謝優化建議!請幫我應用這些建議,生成優化後的版本。
```

### Step 4.2: Platform Customizer Agent - 平台適配 (可選)

Platform Customizer Agent 會提供:

2. **視覺建議** (配圖風格、排版建議)
3. **Hashtag 建議** (5-10 個相關標籤)
4. **發布策略** (最佳時間、頻率)

### Step 4.3: Quality Assurance Agent - 品質檢查

#### 在哪裡操作
打開 "Quality Assurance Agent" Project

#### 輸入模板

```
[最終版本內容]
[從 Step 4.2 複製平台適配後的內容,或從 Step 4.1 複製優化後的內容]

[內容資訊]
- 目標: [如:建立權威]
- 受眾: [如:數位行銷從業者]
- 平台: [如:LinkedIn]

請為這篇內容進行全面的品質檢查,評估:
1. 準確性 (事實正確、數據可靠)
2. 語言品質 (語法、用詞、風格)
3. 邏輯連貫 (結構、論證、過渡)
4. 用戶體驗 (可讀性、視覺、行動引導)
5. 品牌一致性
6. 目標達成度

請給出品質評分和發布建議。
```

#### 預期輸出

Quality Assurance Agent 會提供:
1. **品質評估報告** (6 維度評分、總分、等級)
2. **問題清單** (嚴重程度、位置、修改建議)
3. **發布建議** (是否可發布、修改優先級)
4. **改進方向** (長期品質提升建議)

#### 品質評分標準

- **90+ 分**: 優秀,可直接發布 ✅
- **75-89 分**: 良好,小幅修改後發布 ⚠️
- **60-74 分**: 及格,需要優化 🔄
- **<60 分**: 不及格,需要重寫或大幅修改 ❌

#### 處理品質問題

如果有需要修改的問題:
1. **嚴重錯誤 (🔴)**: 必須修改
2. **一般問題 (🟡)**: 建議修改
3. **優化建議 (🟢)**: 可選改進

根據問題優先級,回到相應的 Step 修改,或直接在 QA Agent 中要求修改。

## 完整範例演示

### 案例背景

**用戶**: Gary,數位行銷顧問
**主題**: 副業變正職的真實路徑
**目標**: 分享經驗,鼓勵想創業的人
**平台**: LinkedIn

---

### Phase 1: Discovery

**Step 1.1 - Master Workflow Agent**

輸入:
```
我想創作一篇關於我從上班族到自由工作者的經歷。

【個人背景】
- 職業: WordPress 開發者 → 小公司負責人
- 經驗: 2018 年開始學習,2020 年全職創業
- 專業: WordPress 網站開發、數位行銷

【創作需求】
- 主題: 副業變正職的真實路徑
- 目標: 分享經驗,鼓勵想創業的人,建立個人品牌
- 受眾: 考慮創業的上班族、想發展副業的人
- 平台: LinkedIn

【個人特質】
- 風格: 實務導向,喜歡分享真實經驗
- 特點: 不迴避困難,坦誠分享失敗和掙扎

請幫我分析並推薦框架。
```

輸出 (摘要):
- 用戶特質: 實務導向型 + 真實坦誠
- 內容目標: 經驗分享 + 個人品牌建立
- 推薦方向: SOLVE, PERSONAL, STORY

**Step 1.2 - Framework Selector Agent**

輸入:
```
[背景資訊]
- 用戶特質: 實務導向型
- 內容目標: 經驗分享、個人品牌建立
- 目標平台: LinkedIn
- 目標受眾: 考慮創業的上班族

[具體需求]
- 主題: 副業變正職的真實路徑
- 個人風格: 實務導向,真實坦誠,不迴避困難
- 期望效果: 提供實用建議,同時引發共鳴

請推薦前 3 名框架並說明理由。
```

輸出 (摘要):
1. **SOLVE Method** (92分) - 實務導向,提供解決路徑
2. **PERSONAL Method** (88分) - 個人品牌建立,真實性
3. **STORY Method** (85分) - 案例展示,經驗分享

選定: **SOLVE Method**

---

### Phase 2: Research

**Step 2.1 - Interview Specialist Agent**

輸入:
```
[背景資訊]
- 選定框架: SOLVE Method
- 內容主題: 副業變正職的真實路徑
- 目標受眾: 考慮創業的上班族

[我的經驗背景]
我從 2018 年開始利用下班時間學習 WordPress,2020 年離職全職創業。過程中遇到很多困難,包括完美主義陷阱、客戶溝通挑戰、收入不穩定等。現在有自己的小公司和員工。

請開始訪談我。
```

## 常見問題解答

### Q1: 一定要完整執行 4 個 Phase 嗎?

**A**: 不一定。你可以根據需求靈活調整:
- **最小流程**: Phase 1 (Master Workflow + Framework Selector) → Phase 3 (Framework Agent)
- **完整流程**: 所有 4 個 Phases
- **重點加強**: 根據需要多次執行某個 Phase

### Q2: 如何在 Agents 間傳遞資訊?

**A**: 手動複製貼上關鍵資訊:
1. 使用結構化格式 (如 `[背景資訊]`、`[訪談素材]`)
2. 只複製關鍵資訊,不需要全部
3. 可以在新對話開始時提供背景

### Q3: 生成的內容不滿意怎麼辦?

**A**: 有幾個選擇:
1. **在同一對話中要求修改**: 提供具體的修改建議
2. **回到 Phase 2 補充素材**: 可能是素材不夠豐富
3. **嘗試不同框架**: 回到 Phase 1 選擇其他推薦框架
4. **手動編輯**: 把 AI 生成的內容作為草稿,手動潤色

### Q4: 訪談階段不知道怎麼回答?

**A**: 參考以下技巧:
- **提供具體細節**: 數字、時間、地點、人物
- **真實坦誠**: 不需要美化,真實的掙扎更有共鳴
- **加入情感**: 不只是事實,也分享當時的感受
- **舉例說明**: 用具體例子支撐觀點

### Q5: 整個流程需要多長時間?

**A**: 視流程完整度:
- **最小流程**: 15-20 分鐘
- **標準流程**: 30-40 分鐘
- **完整流程**: 45-60 分鐘

### Q6: 可以同時創作多篇內容嗎?

**A**: 可以!建議:
1. **分開不同的對話**: 在每個 Agent 的 Project 中開新對話
2. **清楚標記**: 在對話開始時說明是哪篇內容
3. **保存記錄**: 把關鍵資訊記錄在文檔中

### Q7: 生成的內容可以直接發布嗎?

**A**: 建議:
1. **一定要經過 Quality Assurance 檢查**
2. **75+ 分的內容需要小幅修改**
3. **加入個人化元素**: 確保內容有你的聲音
4. **最終人工審核**: AI 是工具,最終決定權在你

### Q8: 如何優化 System Prompt?

**A**: 根據使用經驗:
1. **記錄問題**: 哪些回應不符合預期
2. **調整 Custom Instructions**: 增加或修改指引
3. **提供範例**: 在 Prompt 中加入好的範例
4. **測試驗證**: 修改後測試是否改善
